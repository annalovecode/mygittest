    ## Link MongoDB with Prisma

    <TIL.ItemTags>
      <TIL.ItemTag>Prisma</TIL.ItemTag>
      <TIL.ItemTag>MongoDB</TIL.ItemTag>
    </TIL.ItemTags>

    First to use in MongoDB Atlas, and deploy it to Vercel.
    Whole process:
    1. Prisma Usage:

      1).Project initialization:
      run ```npm init --yes```

      2).Installing Prisma:
          run `yarn add -D prisma` or `npm i -D prisma`

      3).Generating Prisma project configuration:
          run `npx prisma init`

      4).Additionally, executing the above commands will automatically generate the following files:

          prisma/schema.prisma: This is the Prisma schema file used to define your database models and relationships.

          .env file: This is an environment variable file used to store sensitive information and configuration parameters, such as database connection details.

      5).set schema.prisma

          !! important: set .env

          `DATABASE_URL="mongodb+srv://test:test@cluster0.ns1yp.mongodb.net/myFirstDatabase/?retryWrites=true&w=majority"`
          You need to first create a database in MongoDB Atlas, then link to it.

          use `npx prisma db push` to deploy it to MongoDB Atlas

2. How to deploy it to Vercel:

First, click Project Setting - Environment Variables

Then add new Key- Vaule pair:

DATABASE_URL + XXX

Add redeploy it, and it works !

      [Official Docs](https://www.prisma.io/docs/concepts/database-connectors/mongodb)

      [Official Docs](https://www.prisma.io/docs/guides/deployment/deployment-guides/deploying-to-vercel)

      [Reference Docs](https://weihuijieonline.com/post/62f9b260843d301ab3c76f63)

    ## Tailwind CSS !important modifier

    <TIL.ItemTags>
      <TIL.ItemTag>tailwindcss</TIL.ItemTag>
      <TIL.ItemTag>css</TIL.ItemTag>
    </TIL.ItemTags>

    Make any utility important by adding a `!` character to the beginning:

    ```html
    <p class="font-bold !font-medium">
      This will be medium even though bold comes later in the CSS.
    </p>
    ```

    [Official Docs](https://tailwindcss.com/docs/configuration#important-modifier)

    ## :not() pseudo-class

    <TIL.ItemTags>
      <TIL.ItemTag>css</TIL.ItemTag>
      <TIL.ItemTag>selector</TIL.ItemTag>
    </TIL.ItemTags>

    Pake `:not()` ini emang agak tricky, salah penggunaan hasilnya juga bisa ngga sesuai harapan.

    Contoh, styling inline `code` dimana bukan children dari `pre` ataupun `.code-block`. Penggunaannya:

    <TIL.DnD>
      <TIL.Dont>
        ```css {copy:false} {footer:false}
        :not(pre), :not(.code-block) {
          > code {
            background-color: 'red';
          }
        }
        ```
      </TIL.Dont>
      <TIL.Do>
        ```css {copy:false} {footer:false}
        :not(pre, .code-block) {
          > code {
            background-color: 'red';
          }
        }
        ```
      </TIL.Do>
    </TIL.DnD>

    ## Turborepo dependsOn

    <TIL.ItemTags>
      <TIL.ItemTag>turborepo</TIL.ItemTag>
      <TIL.ItemTag>monorepo</TIL.ItemTag>
    </TIL.ItemTags>

    ```json {copy:false} {footer:false}
    "build": {
      "dependsOn": ["^lint", "test"]
    },
    ```

    `dependsOn` dengan ^ menandakan task dari package lain, sedangkan yang tanpa ^ task dari package itu sendiri.

    Misal: app A pas mau build harus running tasks `lint` di semua package dependenciesnya, dan juga harus running task `test` app A itu sendiri.

  </TIL.Item>
